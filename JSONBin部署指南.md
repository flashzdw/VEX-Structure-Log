## 🔐 第一步：注册JSONBin账号

### 1.1 打开注册页面
在浏览器中打开：
```
https://jsonbin.io/register
```

### 1.2 填写注册信息
- 输入**邮箱地址**
- 创建**密码**（至少8位）
- 点击 "Create Account"

### 1.3 验证邮箱
- 打开您的邮箱
- 找到JSONBin发来的验证邮件
- 点击验证链接

### 1.4 登录成功
- 验证后会自动登录
- 看到主界面就成功了！

---

## 📦 第二步：创建数据容器

### 2.1 创建新的Bin
1. 点击蓝色的 **"+ New Bin"** 按钮
2. 在弹出的对话框中：
   - 选择 **"Create a new collection"** 或直接创建bin
   - 在文本框中输入一个初始的空JSON：
     ```json
     []
     ```
   - 这是空数组，表示还没有任何记录

### 2.2 保存Bin
1. 点击 **"Save"** 或 **"Create"** 按钮
2. 创建成功后会跳转到Bin详情页

### 2.3 记录重要信息
在Bin详情页，找到并**复制保存**以下信息：

#### 信息1：Bin ID
- 在URL地址栏能看到，类似：
  ```
  https://jsonbin.io/XXXXXXXX
  ```
- `XXXXXXXX` 就是您的 **Bin ID**
- 复制这个Bin ID

#### 信息2：Access Token（访问令牌）
1. 点击左上角的 **"API Keys"** 或用户头像
2. 找到 **"Default Secret Key"**
3. 点击复制按钮，复制这个密钥
4. 或者创建一个新的Secret Key：
   - 点击 "Create New Key"
   - 输入名称（如：vex-app）
   - 点击 "Submit"
   - 复制生成的密钥

---

## ⚙️ 第三步：配置权限（重要！）

### 3.1 设置Bin权限
在Bin详情页，找到 **"Access Rules"** 或 **"Permissions"** 部分：

1. 设置为 **"Public"** 或 **"Anyone with the link"**
2. 确保 **"Create"** 权限已开启（允许添加数据）
3. 确保 **"Update"** 权限已开启（允许更新数据）
4. 确保 **"Delete"** 权限已开启（允许删除数据）
5. 点击 **"Save"** 保存

### 3.2 重要说明
- JSONBin的免费版默认是私有Bin
- 必须设置为公开或提供正确的API Key才能让前端访问
- 我们会给前端配置API Key，所以即使设为私有也可以访问

---

## 🔧 第四步：更新配置文件

### 4.1 修改 src/config.ts 文件
用文本编辑器打开 `src/config.ts` 文件，替换为您获取的信息：

```typescript
export const JSONBIN_CONFIG = {
  // 替换为您的Bin ID
  binId: 'YOUR_BIN_ID_HERE',
  
  // 替换为您的API Key
  apiKey: 'YOUR_API_KEY_HERE',
  
  // JSONBin API地址（固定不变）
  baseUrl: 'https://api.jsonbin.io/v3'
};
```

## 📤 第六步：部署到腾讯云COS

### 6.1 登录腾讯云COS控制台
打开浏览器访问：
```
https://console.cloud.tencent.com/cos
```

### 6.2 进入存储桶
找到您的存储桶 `vex-structure-log-1429684630`，点击进入。

### 6.3 删除旧文件
选择所有旧文件，点击删除。

### 6.4 上传新文件
1. 点击 **"上传文件"**
2. 选择 `dist` 文件夹中的所有文件：
   - `index.html`
   - `favicon.svg`
3. 点击 **"上传"**

### 6.5 访问网站
1. 进入"基础配置" → "静态网站"
2. 复制"请求域名"
3. 在浏览器打开

## 🆘 如果遇到问题

### 问题1：访问不到数据
- 检查Bin权限是否设置为公开
- 检查API Key是否正确
- 检查Bin ID是否正确

### 问题2：COS上打开空白
- 确认上传的是最新构建的文件
- 尝试强制刷新（Cmd+Shift+R）

### 问题3：数据不更新
- 可能是浏览器缓存，尝试强制刷新
- 或清除浏览器缓存
