## 腾讯云COS部署步骤

### 1. 准备工作

1. 登录[腾讯云控制台](https://console.cloud.tencent.com/cos)
2. 开通对象存储COS服务（首次使用需实名认证）
3. 准备好域名（可选，也可使用COS提供的默认域名）

### 2. 创建存储桶

1. 在COS控制台点击"存储桶列表" → "创建存储桶"
2. 配置存储桶：
   - **名称**：填写一个唯一的名称（如 `vex-structure-log`）
   - **地域**：选择离您最近的地域（如：广州、上海等）
   - **访问权限**：选择"公有读私有写"
   - **其他设置**：保持默认即可
3. 点击"创建"完成存储桶创建

### 3. 配置静态网站

1. 进入刚创建的存储桶
2. 点击左侧菜单"基础配置" → "静态网站"
3. 点击"编辑"，开启静态网站功能
4. 配置：
   - **索引文档**：`index.html`
   - **错误文档**：`index.html`（单页面应用需要）
5. 点击"保存"

### 4. 上传文件

#### 方式一：控制台上传（推荐少量文件）

1. 进入存储桶 → "文件列表"
2. 点击"上传文件"
3. 将 `dist` 目录下的**所有内容**上传（保持目录结构）
4. 确保 `index.html` 在根目录

#### 方式二：使用COSCMD工具（推荐大量文件）

1. 安装COSCMD：
```bash
pip install coscmd
```

2. 配置COSCMD：
```bash
coscmd config -a &lt;SecretId&gt; -s &lt;SecretKey&gt; -b &lt;BucketName&gt; -r &lt;Region&gt;
```

3. 上传文件：
```bash
cd /Users/bbc/Desktop/TRAE-Solo-517/dist
coscmd upload -r ./ /
```

### 5. 访问网站

上传完成后，在"基础配置" → "静态网站"页面可以看到访问地址：
- 默认域名：`https://&lt;BucketName&gt;.cos.&lt;Region&gt;.myqcloud.com`
- 或配置自定义域名后使用您的域名访问

## 配置自定义域名（可选）

1. 进入存储桶 → "域名与传输管理" → "自定义源站域名"
2. 点击"添加域名"
3. 输入您的域名，选择"静态网站源站"
4. 根据提示在DNS服务商处添加CNAME记录
5. 等待DNS生效后即可使用

## 重要提示

### 1. 数据存储说明

本应用数据存储在浏览器的localStorage中，这意味着：
- 数据**不会**上传到服务器或COS
- 更换浏览器或清除浏览器数据会导致记录丢失
- 建议定期使用PDF导出功能备份重要记录

### 2. 性能优化

- 已启用文件压缩（gzip），加载速度快
- 建议在COS控制台开启CDN加速，提升访问速度

### 3. 安全性

- 虽然存储桶设置为"公有读"，但用户数据仅存储在本地，不会泄露
- 建议使用HTTPS访问（COS默认支持）

## 重新部署

当您修改了代码需要重新部署时：

1. 在项目根目录运行构建命令：
```bash
npm run build
```

2. 将新生成的 `dist` 目录内容重新上传到COS

## 常见问题

**Q: 刷新页面出现404错误？**
A: 确保在静态网站配置中，错误文档也设置为 `index.html`

**Q: 数据会保存在服务器上吗？**
A: 不会。所有数据存储在用户浏览器本地，不会上传到任何服务器。

**Q: 如何备份数据？**
A: 使用应用内的"导出PDF"功能，将重要记录导出保存。

**Q: 可以多个用户共享数据吗？**
A: 当前版本不支持。如需团队协作，可以考虑后续添加后端数据库支持。

## 技术支持

如有问题，请检查：
1. 文件是否完整上传
2. 静态网站配置是否正确
3. 浏览器控制台是否有错误信息
