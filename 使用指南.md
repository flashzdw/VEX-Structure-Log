# 📖 VEX 结构记录系统 - 使用指南

## 🚀 快速开始

### 1. 打开应用
- 直接打开 `dist/VEX-结构记录.html` 或访问您部署到腾讯云 COS 的网址

### 2. 添加记录
- 点击右上角的 **"+ 新建记录"**
- 填写日期、负责人、模块、原因、内容等信息
- 可以添加照片
- 点击保存

### 3. 导出数据（团队共享）
- 点击导航栏的 **"导出"** 按钮
- 会下载一个 `.json` 文件
- 把这个文件发送给团队成员

### 4. 导入数据（团队共享）
- 点击导航栏的 **"导入"** 按钮
- 选择别人分享的 `.json` 文件
- 点击导入，数据就会同步过来！

---

## 📋 功能介绍

### ✏️ 记录信息
- **日期** - 记录当天的日期
- **负责人** - 谁负责这次改动
- **模块** - 底盘、抓手、弹射、升降、其他
- **原因** - 为什么要改动
- **内容** - 具体改了什么
- **照片** - 可以上传多张照片
- **测试结果** - 改动后的测试情况
- **遇到的问题** - 过程中遇到的问题
- **下一步计划** - 接下来要做什么

### 📤 导出
- 一键导出所有数据为 JSON 文件
- 文件名自动包含日期：`vex-records-2024-05-20.json`

### 📥 导入
- 选择 JSON 文件导入
- 自动验证数据格式
- 导入成功后自动刷新

### 📄 PDF 导出
- 在 "导出" 页面可以导出为 PDF
- 适合打印和存档

---

## 👥 团队协作方式

### 方式一：主维护人模式（推荐）
1. 指定一个人作为"主维护人"
2. 主维护人负责记录所有数据
3. 定期导出文件分享给团队
4. 其他人可以导入查看最新数据

### 方式二：轮流记录
1. 每个人记录自己负责的内容
2. 导出后互相分享
3. 最后由一个人合并导入

---

## 💾 数据安全

### 本地存储
- 数据保存在浏览器的本地存储里
- 即使刷新页面数据也不会丢失
- 定期导出备份很重要！

### 备份建议
- 每周导出一次数据备份
- 把备份文件存在云端（如腾讯云 COS）
- 重要改动后立即导出备份

## ❓ 常见问题

### Q: 换了电脑数据会丢失吗？
A: 是的。但只要您有导出的备份文件，在新电脑上导入就可以了！

### Q: 可以多人同时编辑吗？
A: 需要协调一下。建议一个人主要维护，其他人导出查看。或者轮流编辑，每次编辑后分享最新的文件。

### Q: PDF 导出乱码怎么办？
A: 这是字体问题。您可以先导出为 JSON 备份，然后用 Chrome 浏览器的打印功能（Ctrl+P）来保存为 PDF。
