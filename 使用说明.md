- ✅ 工程笔记 PDF 导出
- ✅ 中英文双语支持
- ✅ 筛选与搜索
- ✅ 里程碑标记

### 适用场景

- 🏆 VEX 机器人竞赛团队
- 🤖 记录机器人结构迭代
- 📊 工程笔记管理
- 👨‍👩‍👧‍👦 团队协作
- 👀 家长远程查看进度

## 👥 队伍管理

### 创建新队伍

1. 登录后进入设置页面
2. 点击 **"创建新队伍"**
3. 填写队伍名称（如：88888A）
4. 点击 **"创建"**
5. 系统会生成一个**邀请码**

### 加入现有队伍

1. 登录后进入设置页面
2. 点击 **"加入队伍"**
3. 输入队友分享的邀请码
4. 点击 **"加入"**

### 队伍信息

- **队伍 ID**：队伍唯一标识符
- **队伍名称**：自定义的队伍名称
- **邀请码**：用于邀请新成员加入
- **成员列表**：显示所有已加入的队员

## 📝 记录管理

### 新建记录

1. 在首页点击右上角的 **"新建记录"** 按钮
2. 填写记录信息：

#### 基础信息

- **日期**：默认为当天，可修改
- **负责人**：填写你的名字
- **模块**：选择改动的模块（底盘/抓手/弹射/升降/其他）
- **队伍**：选择要记录的队伍（如加入多个队伍）

#### 改动详情

- **改动原因**：为什么要做这次改动？
- **改动内容**：具体改了什么？
- **上传照片**：点击上传按钮，添加改动前后的照片（支持多张）

#### 测试结果

- **测试效果**：改动后的效果如何？
- **遇到问题**：测试中发现了什么问题？
- **下一步计划**：接下来要做什么改进？
- **评分**：给这次改动打分（1-5星）
- **里程碑**：如果是重要版本，勾选此项

3. 点击 **"保存"** 完成

### 查看记录

1. 在首页可以看到所有记录的卡片列表
2. 点击任意记录卡片查看详情
3. 在详情页可以查看完整信息和所有照片

### 编辑记录

1. 打开记录详情页
2. 点击右上角的 **"编辑"** 按钮
3. 修改需要更新的内容
4. 点击 **"保存"** 完成

### 删除记录

1. 打开记录详情页
2. 点击右上角的 **"删除"** 按钮
3. 确认删除操作

⚠️ **注意**：删除操作不可恢复，请谨慎操作！

## 🔍 筛选与搜索

### 搜索记录

1. 在首页顶部的搜索框中输入关键词
2. 系统会自动筛选匹配的记录
3. 可以搜索：
   - 改动内容
   - 改动原因
   - 负责人

### 筛选记录

1. 点击首页的 **"筛选"** 按钮
2. 打开筛选面板
3. 可以按以下条件筛选：
   - **模块**：底盘/抓手/弹射/升降/其他
   - **负责人**：选择特定负责人
   - **队伍**：选择特定队伍

### 清除筛选

- 点击筛选面板的 **"清除所有"** 清除所有筛选条件
- 或点击单个筛选标签的 × 清除特定条件

- ✅ 记录统计
- ✅ 每条记录的完整详情

## ❓ 常见问题

### Q: 为什么登录提示"账号不存在"？

A: 数据库被重置过，需要重新注册账号。

### Q: 为什么注册失败？

A: 检查 PocketBase 服务是否正常运行，以及 users 集合的 Create 规则是否为空。

### Q: 家长访问不了怎么办？

A: 请检查：
1. 花生壳客户端是否在线
2. 映射是否正确配置
3. 您的电脑是否开机并联网
4. 前端和 PocketBase 服务是否正在运行

### Q: 照片上传失败？

A: 检查：
1. 网络连接是否正常
2. 照片大小是否过大（建议每张照片 < 5MB）
3. 浏览器是否支持

### Q: 数据会丢失吗？

A: 数据存储在本地 PocketBase 数据库中，只要不删除数据库文件就不会丢失。建议定期备份。

### Q: 可以在多个设备同时使用吗？

A: 可以！多人可以同时登录，数据会实时同步。

### Q: 如何备份数据？

A: 备份 `pb_data` 文件夹即可，该文件夹包含所有数据。

### PocketBase 管理后台

访问：`http://localhost:8090/_/`

- 账号：`admin@example.com`
- 密码：`password123`

可以在此管理：
- 用户账号
- 数据记录
- API 权限规则
- 数据库备份

### API 权限规则配置

确保所有集合的权限规则正确：

#### users 集合

- List/Search: `id = @request.auth.id`
- View: `id = @request.auth.id`
- Create: **留空**（允许注册）
- Update: `id = @request.auth.id`
- Delete: `id = @request.auth.id`

#### records 集合

- List/Search: `@request.auth.id != ""`
- View: `@request.auth.id != ""`
- Create: `@request.auth.id != ""`
- Update: `@request.auth.id != ""`
- Delete: `@request.auth.id != ""`

#### teams 集合

- List/Search: `@request.auth.id != ""`
- View: `@request.auth.id != ""`
- Create: `@request.auth.id != ""`
- Update: `@request.auth.id != ""`
- Delete: `@request.auth.id != ""`

#### team_members 集合

## 📊 使用技巧

### 记录的最佳实践

1. **及时记录**：每次改动后立即记录，避免遗忘细节
2. **详细描述**：尽可能详细地描述改动内容和测试结果
3. **上传照片**：一图胜千言，多拍照片辅助说明
4. **标记里程碑**：重要版本标记为里程碑，方便快速回顾
5. **合理评分**：客观评价每次改动的效果

### 团队协作建议

1. **固定负责人**：每个模块可以有专门的负责人
2. **定期回顾**：每周一起回顾所有记录
3. **统一规范**：约定好记录的格式和用语
4. **互相审核**：队友之间可以互相查看和补充记录
